# CLAUDE.md — конституция репозитория Fin_Capital_2026

Этот файл — правила игры для ассистента (и для людей-коллабораторов). Он
описывает, что это за репозиторий, как устроена `wiki/` и чего здесь делать
**нельзя**. Цель — чтобы любой следующий сеанс с Claude Code мог с нуля понять
состояние проекта и продолжить работу, не ломая дисциплину.

## 1. Что это за репозиторий

Это рабочая база знаний по материалам Telegram-канала «Капитал» (SGCapital).
Источник — официальный HTML-экспорт канала:

- `messages.html`, `messages2.html`, `messages3.html` — сырые экспортированные
  сообщения (1996 постов за 2017-04-04 — 2026-04-14).
- `css/`, `js/`, `files/`, `images/`, `photo_*.jpg` — служебные файлы того же
  экспорта. Их не трогаем.

Поверх этого построено два слоя:

1. **Корпус и индекс** (`tools/chat.jsonl`, `tools/parse_chat.py`,
   `tools/build_index.py`, `INDEX.md`) — машиночитаемый JSONL и
   автогенерируемый тематический указатель по ключевым словам. Это «сырая
   руда».
2. **Wiki** (`wiki/`) — дистиллят. Ручной синтез принципов канала в
   эвергрин-страницы, каждая со ссылками на первоисточники в `chat.jsonl`.

`INDEX.md` — это каталог сообщений. `wiki/` — это **что мы из этих сообщений
выносим**. Это разные артефакты, и их нельзя путать.

## 2. Правила `wiki/`

### 2.1. Язык и аудитория

- Язык страниц — русский (источник — русскоязычный).
- Аудитория — человек, который хочет понять инвестиционную философию канала
  без необходимости читать 1996 постов.

### 2.2. RU-контент исключён из таксономии

Wiki **намеренно** не покрывает Россия-специфичные темы:

- российский рынок акций (Мосбиржа, отдельные эмитенты РФ);
- ОФЗ и рублёвые облигации;
- курс рубля, валютное регулирование РФ;
- ИИС, НДФЛ и российские налоговые льготы;
- геополитика и санкции.

Причины: (а) эти темы быстро устаревают и сильно привязаны к регуляторному
контексту, (б) канал начинался как эвергрин-ресурс и большая часть ценности в
универсальных принципах. Сообщения по этим темам остаются в `chat.jsonl` и
`INDEX.md`, но в `wiki/` не попадают.

Таксономия эвергрин-разделов:

```
wiki/
├── overview.md           # что это и как читать
├── index.md              # автогенерируемый каталог страниц
├── log.md                # журнал ingestion-сессий
├── principles/           # инвестиционная философия
│   ├── passive-vs-active.md
│   ├── commissions.md
│   ├── long-horizon.md        # (резерв, пока не написана)
│   └── risk-and-drawdowns.md  # (резерв)
├── portfolio/            # построение портфеля
│   ├── asset-allocation.md    # (резерв)
│   ├── rebalancing.md
│   └── diversification.md     # (резерв)
├── assets/               # классы активов
│   ├── equities-global.md     # (резерв)
│   ├── bonds-global.md        # (резерв)
│   ├── gold.md
│   └── crypto.md              # (резерв)
├── behavior/             # психология инвестора
│   ├── biases.md              # (резерв)
│   └── advisors-and-products.md  # (резерв)
├── macro/                # макро-эвергрин
│   ├── crises-and-recessions.md  # (резерв)
│   └── inflation-and-rates.md    # (резерв)
└── _sources/             # автогенерируемые списки первоисточников
    └── <topic>.md
```

«Резерв» означает: слот в таксономии есть, страницу **не создаём как
stub**. Страница появляется только тогда, когда у кого-то есть синтез, чтобы
её наполнить.

### 2.3. Что такое «страница» в `wiki/`

Каждая страница (кроме overview/index/log/`_sources/`) строится по одному
шаблону:

```markdown
# <Заголовок темы>

> Короткая формулировка в 1–2 предложения: что канал говорит по этой теме.

## TL;DR
- 3–6 тезисов.

## Развёрнутая позиция
Связный текст-синтез на 2–6 абзацев. Это **не** цитатник — это дистилляция.

## Цитаты-опоры
1–3 прямые цитаты из чата, сокращённые до ключевого места, каждая с датой и
ссылкой `chat:<msg_id>`.

## Первоисточники
Ссылка на `_sources/<topic>.md` с полным списком релевантных сообщений,
сгенерированным `tools/build_wiki.py`.

## См. также
Ссылки на соседние страницы wiki.
```

Формат ссылок на сообщения канала:
- `chat:messageNNN` — человекочитаемая форма (идентификатор из `chat.jsonl`).
- В `_sources/` идут рабочие ссылки вида `../messages.html#messageNNN`,
  которые открываются при клонировании репозитория.

### 2.4. Что в `wiki/` не попадает

- **Сырые дампы на русском.** Нельзя копипастить сообщения целиком. Можно
  короткую (1–3 предложения) цитату как опору для тезиса — не больше.
- **Страницы-заглушки.** Если нечего сказать — слот в таксономии пустует до
  следующего сеанса.
- **Тикеры/цены/прогнозы дня.** Wiki — про принципы, не про календарь.
- **Личные реквизиты и любая PII** из чата, если бы она там обнаружилась.
- **Россия-специфичные темы** из списка выше.

## 3. Инструменты

### `tools/parse_chat.py`
Парсит `messages*.html` в `tools/chat.jsonl`. Запускать при обновлении
экспорта канала.

### `tools/build_index.py`
Генерирует `INDEX.md` — тематический указатель по ключевым словам. Покрывает
**все** темы, включая RU.

### `tools/build_wiki.py`
Генерирует:

1. `wiki/_sources/<topic>.md` — для каждой темы из встроенной таблицы
   `WIKI_TOPICS` (только эвергрин-темы, RU исключён) пишет хронологический
   список сообщений, совпавших по ключевым словам.
2. `wiki/index.md` — каталог, собранный сканированием `wiki/**/*.md`:
   извлекает H1 и первую строку после него как краткое описание. Порядок
   определяется структурой каталогов.

**Что скрипт НЕ делает:**
- не трогает ручные страницы (`principles/`, `portfolio/`, `assets/`,
  `behavior/`, `macro/`, `overview.md`, `log.md`);
- не переписывает `CLAUDE.md`;
- не пишет никаких прогнозов, текста или «резюме» — только структурированные
  списки/каталог.

Запуск:
```
python3 tools/build_wiki.py
```

## 4. Процесс ingestion

Каждый раз, когда добавляем материал в `wiki/`:

1. Работаем на ветке вида `claude/<task>-<tag>`, не в main.
2. Ручные страницы — сначала синтез, потом цитаты, потом ссылка на
   `_sources/<topic>.md`.
3. Запустить `python3 tools/build_wiki.py`, чтобы обновились `_sources/` и
   `index.md`.
4. Добавить запись в `wiki/log.md` (дата, автор, одно-двухстрочный summary
   изменений, список тронутых страниц).
5. Один коммит = один логический шаг. Сообщения коммита на английском,
   сжатые.

## 5. Пределы автономии ассистента

Claude может **свободно**:
- Читать любые файлы в репо.
- Править файлы в `wiki/`, `tools/`, создавать новые страницы в таксономии из
  §2.2.
- Перезапускать `tools/build_index.py`, `tools/build_wiki.py`.
- Коммитить и пушить на назначенную рабочую ветку.

Claude **обязан спросить** перед:
- Созданием новой темы верхнего уровня в таксономии.
- Любыми изменениями в `messages*.html`, `tools/chat.jsonl`,
  `tools/parse_chat.py` (это первоисточники).
- Мержами и force-push.
- Изменениями, которые удаляют уже написанный синтез-контент.

Claude **не должен**:
- Копировать большие куски сообщений канала как тело страницы.
- Добавлять страницы по RU-темам из §2.2.
- Предсказывать/прогнозировать — всё, что в wiki, должно опираться на уже
  сказанное в канале.
